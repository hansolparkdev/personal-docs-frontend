# 기획서: api-integration

## Meta
- type: feature
- package: frontend
- design-ref: none

## Problem
현재 프론트엔드는 로그인·파일·사이드바·챗 화면이 모두 목업 데이터로 동작한다. 사용자는 실제로 계정을 만들 수 없고, 업로드한 문서가 저장되지 않으며, 로그인 여부와 무관하게 모든 페이지에 접근할 수 있어 제품으로서 기능하지 않는다. FastAPI 백엔드(`/api/v1`)는 인증·파일 영역 API가 준비되어 있으므로, 목업을 실제 백엔드에 연결해 제품을 동작 가능한 상태로 전환해야 한다.

## Users (사용자 / 맥락)
- **누구**: 개인 지식·문서를 축적하려는 1인 최종 사용자.
- **맥락**: 웹 브라우저에서 본인 계정으로 로그인하여 사적 문서를 올려두고 관리함.
- **원하는 것**: 실제 계정·세션으로 자신의 문서 자산에 안전하게 접근·유지하는 것.

## Goals
- 사용자가 실제 계정으로 회원가입·로그인·SSO를 완료하고, 새로고침 후에도 세션이 유지된다.
- 사용자가 업로드한 문서가 서버에 보관되고 목록·삭제·다운로드가 실제 데이터로 동작한다.
- 비로그인 사용자는 보호된 화면에 진입할 수 없고, 로그인 사용자는 공개 화면에서 자동으로 앱으로 진입한다.

## Non-goals
- 챗(RAG) API는 백엔드에 아직 없으므로 이번 범위에 포함하지 않는다. 챗 화면은 목업 상태를 유지한다.
- 신규 UI/화면 추가, 디자인 변경, 토큰 리프레시 외 고급 보안 정책(세션 만료 UX 고도화 등)은 다루지 않는다.
- 관리자 기능, 파일 공유, 파일 처리(인덱싱) 상태 표시 UX 확장은 범위 외.

## Capabilities
- `auth-integration` — 회원가입·로그인·SSO 콜백·토큰 저장 및 자동 갱신·현재 사용자 정보 조회. 사이드바 유저 푸터와 로그아웃을 실제 사용자 데이터·세션 종료로 대체한다.
- `file-integration` — 파일 업로드(용량·확장자 제한 반영), 내 파일 목록 조회, 삭제, 다운로드 URL 획득. 업로드 결과가 목록에 반영되는 흐름을 실제 데이터로 대체한다.
- `session-guard` — 보호 라우트(`(app)`)와 공개 라우트(`(public)`)의 접근 제어를 실제 세션 상태 기반으로 동작시킨다. 미인증 시 로그인 화면으로 유도, 인증 시 공개 화면에서 앱으로 유도한다.

## UX Flow (사용자 흐름)

### 정상 흐름
1. 비로그인 사용자가 `/login`에서 가입 또는 로그인하거나 SSO로 진입한다.
2. 인증 성공 시 앱 홈으로 이동하고, 사이드바에 본인 정보가 표시된다.
3. `/docs`에서 문서를 업로드하면 목록에 즉시 반영되고, 삭제·다운로드가 동작한다.
4. 새로고침·재방문 시 세션이 유지되며 보호 화면에 바로 접근할 수 있다.
5. 로그아웃 시 세션이 종료되고 로그인 화면으로 돌아간다.

### 예외 흐름
- 잘못된 자격증명·중복 가입·용량/확장자 위반 등 백엔드 오류는 화면에서 사용자가 이해할 수 있는 메시지로 안내된다.
- 토큰 만료 시 사용자는 중단 없이 계속 사용할 수 있어야 하며, 갱신 실패 시 로그인 화면으로 유도된다.
- 챗 화면은 백엔드 미제공 상태임을 사용자가 인지할 수 있어야 한다(목업 유지 + 안내).
